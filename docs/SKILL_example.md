# Document Summarizer

## Goal
将长文档（PDF、DOCX、Markdown、TXT）压缩成结构化摘要，帮助用户快速获取文档核心内容，节省阅读时间。专注于提取和呈现信息，不进行分析、评估或建议。

## Decision Tree
1. **输入验证**：
   - 用户是否提供了文档或文档内容？
   - 文档格式是否支持（PDF/DOCX/Markdown/TXT）？
   - 文档大小是否合理？

2. **边界检查**：
   - 用户请求是否属于摘要任务（总结、提炼、概括）？
   - 是否包含分析、评估、建议等超出摘要范围的任务？
   - 如果是混合请求，明确告知技能边界

3. **处理类型判断**：
   - 如果是完整文档 → 生成全面摘要
   - 如果是会议纪要 → 提取行动项和决策点
   - 如果是技术文档 → 提取核心概念和步骤
   - 如果是报告 → 生成执行摘要

4. **摘要生成**：
   - 提取核心信息
   - 结构化组织（概述、关键要点、结论、行动项）
   - 控制摘要长度（原文的10-20%）

5. **输出验证**：
   - 摘要是否包含文档核心内容？
   - 结构是否清晰？
   - 长度是否适中？
   - 是否保持信息中立性（不添加分析或评估）？

## Workflow
1. **接收输入**：确认文档内容和格式
2. **边界确认**：验证请求属于摘要任务范围
3. **解析文档**：提取文本内容，识别结构
4. **内容提取**：识别关键信息、主题、结论
5. **摘要生成**：根据文档类型生成结构化摘要
6. **质量验证**：检查摘要完整性、准确性、中立性
7. **输出交付**：提供最终摘要，明确说明技能边界

## Constraints
- 只处理文档摘要类任务，不做翻译、改写、问答
- 不进行分析、评估、批判、建议
- 输出必须结构清晰，包含核心要点
- 摘要长度适中，不超过原文30%
- 保持原文关键信息和上下文，不添加主观分析
- 不进行复杂决策分析或内容创作

## Validation
- **完整性**：摘要包含文档核心信息
- **结构性**：结构清晰（标题/要点/结论/行动项）
- **长度控制**：摘要长度适中（原文的10-20%）
- **准确性**：保持原文关键信息和上下文
- **中立性**：不添加分析、评估或建议
- **边界清晰**：明确区分摘要任务和分析任务
